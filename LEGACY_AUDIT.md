## Поиск прежнего репозитория в интернете

Поиск по GitHub/вебу по строкам «ShareTube telegram bot youtube downloader», уникальным строкам скрипта и username бота `@sharetube_bot` однозначного репозитория **не нашёл** — найдены только чужие одноимённые/похожие проекты (ytdlbot и т.п.), код которых не совпадает с локальным скриптом. Согласно заданию, за основу взят локальный скрипт.

## Карта поведения старого скрипта (перенесено в новую архитектуру)

1. Принимает ссылку YouTube (`youtube.com`/`youtu.be`) в личке и группах → **новый:** любой поддерживаемый источник, детект по нормализованному URL.
2. Редактируемое сообщение прогресса с прогресс-баром, троттлинг правок 1.5 с → **перенесено** (этапы + бар + скорость + ETA).
3. Формат `bv*[height<=1080][ext=mp4]+ba[ext=m4a]/...` → **перенесено** как пресет «1080p» и логика «MP4 без перекодирования предпочтительнее».
4. Если файл < 50 МБ — отправка как video, иначе — ссылка на файл → **перенесено** и обобщено: конфигурируемые лимиты, Local Bot API, подписанные ссылки с TTL.
5. Лимит папки (3 ГБ в .bak, 15 ГБ в финале) с удалением самых старых файлов → **перенесено** в LocalStorageProvider (`MAX_STORAGE_GB`).
6. cookies.txt рядом со скриптом подхватывался автоматически → **перенесено** в cookie profiles (защищённое хранение, не в Git).
7. Прокси для yt-dlp (закомментирован) + `NO_PROXY` для локальных адресов → **перенесено** в OutboundProfile (HTTP proxy + Xray) с обязательной маршрутизацией.
8. Восстановление polling после сетевых ошибок → **перенесено** (PTB + restart policy + идемпотентность).
9. Домен `sharetube.appswire.ru` уже использовался для ссылок → сохранён как production-домен.

## Проблемы старого кода (устранены в новой версии)

- Bot token, IP, ID администратора захардкожены в коде (в т.ч. в бэкапе — второй токен и пароли прокси). Теперь: только env/secrets, маскирование в логах.
- `os.system("sudo systemctl restart ...")` — command injection surface; убрано.
- Гонки: глобальный `temp.%(ext)s` для всех загрузок — параллельные скачивания затирали друг друга. Теперь: отдельный временный каталог на задание.
- Имя файла по `len(existing_files)+1` — коллизии. Теперь: opaque tokens.
- Flask `send_from_directory` без подписи и TTL — публичный листинг файлов по угадываемым именам. Теперь: подписанные одноразовые ссылки.
- Блокирующий однопоточный polling: одна загрузка блокировала бота. Теперь: очередь фоновых задач (ARQ) + отдельные worker-контейнеры.
- Нет таймаутов subprocess, нет отмены, нет лимитов размера/длительности, нет SSRF-защиты, нет БД, нет повторной отправки по file_id.
- Сообщение об ошибке отдавало пользователю сырой traceback (`str(e)`).

## Что сохранено

- Название **ShareTube**, бот **@sharetube_bot**, домен **sharetube.appswire.ru**.
- Русскоязычный UX бота, стиль прогресс-сообщений (бар/скорость/ETA).
- Формат-стратегия «до 1080p, MP4+M4A без перекодирования».
- Схема «маленький файл → в Telegram, большой → ссылка», лимит хранилища с вытеснением старых файлов.
- cookies.txt как источник для cookie profile YouTube (переносится администратором вручную, не через Git).

Оригиналы (`main.py`, `main.py.bak`, `proxy.py`) оставлены нетронутыми в корне рабочей папки и исключены из Git; в `legacy/` лежат копии с отредактированными секретами.
